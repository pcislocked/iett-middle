- [ ] Fix un-debounced external API calls in `routers/routes.py` `get_route_buses`: The endpoint directly calls `ntcapi_client.get_route_buses_ybs` bypassing the standard `cache_get_or_fetch`. This triggers N external network requests for N concurrent incoming requests (DoS vector).
- [ ] Fix missing cache for negative responses in `mobiett_client.py` `get_hat_id`: The function returns `None` for invalid routes without caching the miss, allowing an attacker to bypass cache by repeatedly querying invalid route codes and triggering unbounded upstream API calls.
- [ ] Fix Thundering Herd vulnerability in `cache.py` `cache_get_or_fetch`: If `fetcher()` raises an exception, the lock is released without caching an error, causing all waiting requests to sequentially become the leader and hit the failing backend one by one.
- [ ] Fix background task fragility in `cache.py` `sweep_forever`: The loop lacks a `try...except Exception:` block. Any unhandled exception will permanently crash the sweeper, causing cache memory to grow unbounded.
- [ ] Fix incorrect error propagation logic in `iett_client.py` `get_route_buses`: If `json_res` fails but `soap_res` legitimately returns an empty list `[]` (meaning no buses are active), the code incorrectly raises `json_res` instead of returning the empty list.
- [ ] Fix unhandled `get_stop_detail` exceptions in `iett_client.py` and `routers/stops.py`: If both SOAP and JSON fail, `iett_client.py` raises the `json_res` exception, which is not caught in `routers/stops.py` `_fetch()`, resulting in a bare 500 error instead of a graceful 502/503 response.
